lhyin发布文章及提交github和oschina备忘录

2017-04-01 阅读(ing...)

代码源

github(本地镜像 myblog_gh): https://github.com/Alec-Yin/Alec-Yin.github.io
oschina(本地镜像 myblog_os): http://git.oschina.net/yinluhui/myblog

写文章

打开myblog_gh根目录源码
本地一般有两个代码镜像，一个是github，一个是oschina。写文章时，需要在github中写，然后把修改后的文章，拷贝到oschina文件中，提交。

git Bash或Cmder定位到myblog_gh根目录
这里是hexo初始化环境的目录，编辑、编译、发布都要在此目录。

创建博客文章

  • 技术类

    $ hexo new it “文章名称”
    或
    $ hexo new “文章名称” //因为_config.yml的default_layout默认为 it 类型博客

说明：文章会在source/_posts 目录下生成名为【it-文章名称.md】的文件，直接在里面编辑就行了。以下雷同。

  • 语文课文

    $ hexo new article “文章名称”

  • 诗词

    $ hexo new poetry “文章名称”

部署及发布

监控修改

$ hexo s -g
$ hexo server -generate

说明：hexo generate是生成静态页面；hexo server是启动本地调试（http://localhost:4000）

部署并提交代码
每次部署的步骤，可按以下三步来进行。

$ hexo clean //清除缓存文件 (db.json) 和已生成的静态文件 (public)
$ hexo generate （==$ hexo g）//生成静态文件
$ hexo deploy （==$ hexo d）//部署网站

更多命令：https://hexo.io/zh-cn/docs/commands.html

同步github和oschina代码

把刚才myblog_gh下的文件拷贝到myblog_os中，只拷贝业务文件夹即可，去除.deploy_git、node_modules、.git文件。
